在vue-cli脚手架里的 config 文件夹下的index.js文件里面 单单改了这个 assetsPublicPath: './' 还是单单不够的。 倘若你的src文件夹里面的main.js(项目入口文件里),路由配置模式 设置为 mode: 'history' 这样子的话，打包出来的dist目录下的index页面也是本地直接访问不了的。 所以若想在本地直接打开打包后的index页面可以将路由配置模式： mode: 'history'给注释掉。 官网的原文解释： 不过这种模式要玩好，还需要后台配置支持。因为我们的应用是个单页客户端应用， 如果后台没有正确的配置， 当用户在浏览器直接访问 http://oursite.com/user/id 就会返回 404，这就不好看了。
