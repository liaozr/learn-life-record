在项目会有需要使用全局变量的需求，来处理一些频繁的操作，大家都应该会绑定到window对象上， 但是这种方式不适合服务端渲染，因为服务端没有 window 对象, 是 undefined, 当试图去访问属性时会报错.我总结了两个靠谱的方法 代理到Vue的原型对象 由于所有的组件都会从 Vue 的原型对象上继承它们的方法, 因此我们只要 Object.defineProperty(Vue.prototype, '$xxx', { value: xxx }); 就可以在所有组件/实例中通过 this.$xxx: 的方式访问插件了~而不需要定义全局变量或者手动的引入了~ 至于为什么要用Object.defineProperty这个方法， 是因为通过Object.defineProperty绑定的属性是只读的， 以防一起开发项目的协(zhu)作(dui)者(you)去重写或者覆盖该方法的值。 vuex大法 vuex的出现就是vue为了集中式存储管理应用的所有组件的状态， 所以说全局变量和方法都可以放到vuex当中~具体用法就不加阐述了，大家可仔细阅读vuex文档
